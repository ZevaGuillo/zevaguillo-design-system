# Modal Component

El componente `Modal` es un diálogo que aparece sobre el contenido principal, requiriendo interacción del usuario antes de continuar.

---

## Especificaciones del Componente

### Propiedades

| Prop        | Tipo                             | Default    | Descripción                    |
| ----------- | -------------------------------- | ---------- | ------------------------------ |
| `open`      | `boolean`                        | `false`    | Controla visibilidad del modal |
| `title`     | `string`                         | -          | Título del modal               |
| `children`  | `ReactNode`                      | -          | Contenido del modal            |
| `size`      | `"small" \| "medium" \| "large"` | `"medium"` | Tamaño del modal               |
| `onClose`   | `() => void`                     | -          | Callback al cerrar             |
| `className` | `string`                         | `""`       | Clases CSS adicionales         |

### Dimensiones

| Size     | Max Width |
| -------- | --------- |
| `small`  | 400px     |
| `medium` | 600px     |
| `large`  | 800px     |

---

## Cuándo Usar Modal

### ✅ Usar Modal cuando:

1. **Confirmaciones importantes**: El usuario debe confirmar una acción destructiva
2. **Edición de datos**: Formularios que requieren concentración
3. **Información crítica**: Alertas que no pueden ignorarse
4. **Selección única**: Cuando se necesita elegir una opción de una lista
5. **Flujos largos**: Procesos de varios pasos

### ❌ NO usar Modal cuando:

1. **Información simple**: Para mostrar mensajes cortos, usa `Toast` o `Alert`
2. **Navegación**: Para ir a otra página, usa `Link`
3. **Contenido extenso**: Si hay mucho contenido, considera usar una página
4. **Acciones frecuentes**: Si el usuario usa la función frecuentemente, una página es mejor

## Guía de Tamaños

### Small

```tsx
<Modal open={open} onClose={close} size="small" title="Confirm">
  <p>Are you sure?</p>
</Modal>
```

**Cuándo usarlo:**

- Confirmaciones simples
- Mensajes breves
- Alertas de confirmación

---

### Medium (Predeterminado)

```tsx
<Modal open={open} onClose={close} title="Edit Profile">
  <Form />
</Modal>
```

**Cuándo usarlo:**

- Formularios de edición
- Contenido moderado
- La mayoría de los casos de uso

---

### Large

```tsx
<Modal open={open} onClose={close} size="large" title="Create Report">
  <ComplexForm />
</Modal>
```

**Cuándo usarlo:**

- Formularios complejos
- Múltiples secciones
- Contenido con mucho espacio

## Mejores Prácticas

### 1. Siempre proporciona onClose

```tsx
// ✅ CORRECTO
const [open, setOpen] = useState(false);
<Modal open={open} onClose={() => setOpen(false)}>
  Content
</Modal>

// ❌ INCORRECTO - Sin forma de cerrar
<Modal open={open}>
  Content
</Modal>
```

### 2. Título claro y descriptivo

```tsx
// ✅ CORRECTO
<Modal title="Delete Account">
  <p>Are you sure?</p>
</Modal>

// ❌ INCORRECTO
<Modal title="Warning">
  <p>Are you sure?</p>
</Modal>
```

### 3. Cerrar con Escape y click outside

El componente ya incluye estas funcionalidades por defecto.

```tsx
// ✅ Ya viene incluido
<Modal open={open} onClose={close}>
  {/* Se cierra con ESC y click fuera */}
</Modal>
```

### 4. Focus management

El modal trapping focus automáticamente dentro del contenido.

```tsx
// ✅ El foco se maneja automáticamente
<Modal open={open} onClose={close}>
  <Button>First</Button>
  <Button>Second</Button>
  <Input />
</Modal>
```

### 5. body scroll lock

El scroll del body se bloquea automáticamente cuando el modal está abierto.

```tsx
// ✅ Scroll bloqueado automáticamente
<Modal open={open} onClose={close}>
  <p>No accidental scrolling!</p>
</Modal>
```

## Malas Prácticas

### 1. Modal sin título necesario

```tsx
// ❌ INCORRECTO - Confuso
<Modal open={open} onClose={close}>
  <p>Your changes have been saved</p>
</Modal>

// ✅ CORRECTO - Usar Alert o Toast
<Alert>Your changes have been saved</Alert>
```

### 2. Contenido muy largo

```tsx
// ❌ INCORRECTO - Scrolling infinito
<Modal open={open} onClose={close} title="Terms">
  <div style={{ height: "2000px" }}>Very long content...</div>
</Modal>

// ✅ CORRECTO - Nueva página
// Crear una ruta /terms para contenido extenso
```

### 3. Múltiples modals anidados

```tsx
// ❌ INCORRECTO
<Modal open={open} onClose={close}>
  <Modal open={innerOpen} onClose={() => setInnerOpen(false)}>
    Nested!
  </Modal>
</Modal>

// ✅ CORRECTO - Un solo modal con pasos
<Modal open={open} onClose={close}>
  {step === 1 && <Step1 />}
  {step === 2 && <Step2 />}
</Modal>
```

### 4. Acciones fuera del modal

```tsx
// ❌ INCORRECTO - Confuso
<div>
  <Modal open={open} onClose={close} title="Edit">
    <Form />
  </Modal>
  <Button>Save</Button> {/* Fuera del modal */}
</div>

// ✅ CORRECTO
<Modal open={open} onClose={close} title="Edit">
  <Form />
  <div style={{ display: 'flex', justifyContent: 'flex-end', gap: '0.5rem' }}>
    <Button variant="ghost">Cancel</Button>
    <Button variant="primary">Save</Button>
  </div>
</Modal>
```

## Implementación

### Básico

```tsx
import { useState } from "react";
import { Modal, Button } from "@zevaguillo/ui-components";

function Example() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button onClick={() => setOpen(true)}>Open Modal</Button>
      <Modal open={open} onClose={() => setOpen(false)} title="Modal Title">
        <p>Modal content here</p>
      </Modal>
    </>
  );
}
```

### Confirmación

```tsx
import { useState } from "react";
import { Modal, Button } from "@zevaguillo/ui-components";

function ConfirmDelete() {
  const [open, setOpen] = useState(false);

  const handleConfirm = () => {
    deleteItem();
    setOpen(false);
  };

  return (
    <>
      <Button variant="danger" onClick={() => setOpen(true)}>
        Delete
      </Button>
      <Modal open={open} onClose={() => setOpen(false)} title="Delete Item">
        <p>
          Are you sure you want to delete this item? This action cannot be
          undone.
        </p>
        <div
          style={{
            display: "flex",
            justifyContent: "flex-end",
            gap: "0.5rem",
            marginTop: "1rem",
          }}
        >
          <Button variant="ghost" onClick={() => setOpen(false)}>
            Cancel
          </Button>
          <Button variant="danger" onClick={handleConfirm}>
            Delete
          </Button>
        </div>
      </Modal>
    </>
  );
}
```

### Formulario

```tsx
import { useState } from "react";
import { Modal, Input, Button } from "@zevaguillo/ui-components";

function EditModal() {
  const [open, setOpen] = useState(false);
  const [name, setName] = useState("");

  const handleSave = () => {
    updateName(name);
    setOpen(false);
  };

  return (
    <>
      <Button onClick={() => setOpen(true)}>Edit</Button>
      <Modal open={open} onClose={() => setOpen(false)} title="Edit Name">
        <div style={{ display: "flex", flexDirection: "column", gap: "1rem" }}>
          <Input
            label="Name"
            value={name}
            onChange={setName}
            placeholder="Enter name"
          />
          <div
            style={{
              display: "flex",
              justifyContent: "flex-end",
              gap: "0.5rem",
            }}
          >
            <Button variant="ghost" onClick={() => setOpen(false)}>
              Cancel
            </Button>
            <Button variant="primary" onClick={handleSave}>
              Save
            </Button>
          </div>
        </div>
      </Modal>
    </>
  );
}
```

## Accesibilidad

### Características incluidas

1. **Focus trap**: El foco no puede salir del modal
2. **Cerrar con ESC**: Presionar Escape cierra el modal
3. **Click fuera**: Click en el overlay cierra el modal
4. **Rol ARIA**: Se usa `role="dialog"` y `aria-modal="true"`
5. **Label**: El título se associa con `aria-labelledby`

### Requisitos adicionales

- ✅ El título debe ser descriptivo
- ✅ El primer elemento enfocable debe tener foco automáticamente
- ✅ El foco debe retornar al trigger al cerrar
